## What was done this session

### PR #3 Review & Merge

Used `scout` + `researcher` subagents to analyze GitHub PR #3 (`atomlab/feature/env-variable-support`). Found and fixed 2 medium-severity bugs before merging.

**Scout findings:**
| Severity | Issue |
|----------|-------|
| 🔴 HIGH | ALL_CAPS regex silently breaks literal keys matching `^[A-Z][A-Z0-9_]*$` pattern |
| 🟡 MEDIUM | `commandValueCache` never invalidates — stale keys after rotation |
| 🟡 MEDIUM | `execSync` has no timeout |
| 🟡 MEDIUM | Auto-enable via env var — can't opt-out if backend config missing (actually safe) |
| 🟢 LOW | SearXNG not in `FALLBACK_ENV_MAP` (intentional) |
| 🟢 LOW | `getKeySource` wrong type for literal ALL_CAPS keys |

**Research findings:** Ecosystem convention (OpenClaw, pi-web-providers) treats ALL_CAPS strings as env var references. The ALL_CAPS breaking change is consistent with this — real API keys (sk-, hex, Bearer) don't match the pattern. Documented in README.

**Reviewer fixes applied:**
- `execSync` timeout: `COMMAND_TIMEOUT_MS = 5_000` (5s)
- `clearCredentialCache()` added; called in `refreshConfig()` (key rotation works)
- Backend dispatcher fully migrated to `resolveBackendKey()` (all 9 auth backends)
- `search-status` updated to show key source via `getKeySource()`
- `MISSING_KEY_HELP` updated to describe all 4 credential mechanisms
